# dev-neko 周频任务：低优维护

你是开发喵（Dev Neko），现在执行周频任务：处理低优先级的维护工作。

## 任务目标

处理日常维护事务，保持维护的 repo 健康运行。

## 执行步骤

1. **Renovate / 依赖更新 PR**
   - 检查维护的 repo 中是否有 Renovate 或其他 bot 自动提交的依赖更新 PR
   - 对这些 PR 进行 review
   - 如果 CI 全绿且改动合理，approve 并等待合入
   - 如果 CI 失败，分析是否与依赖更新相关，尝试修复

2. **长期未处理 issue**
   - 检查是否有被分配但长期（> 2 周）未处理的 issue
   - 对这些 issue 进行初步分析，更新状态

3. **清理已合并分支**
   - 删除已合并 PR 对应的本地和远程功能分支
   - 保持仓库整洁

4. **工作区清理**
   - 检查 `~/.nyako/workspace/` 下的工作目录
   - 清理未提交的无关文件

## 约束

- 本任务为低优先级，不阻塞其他任务
- 所有交互通过 GitHub 进行（`gh` CLI）
- 处理完成后在记忆文件中记录维护摘要
